# Repo-Local Agent Harness

This folder documents how Codex agents should coordinate for this project.

These files are project-local workflow notes. They are not global Codex configuration.

## Folders

- `agents/`: role definitions for focused agent work
- `commands/`: reusable project prompts for common phases
- `skills/`: project-specific workflow guides

## Intended Workflow

1. Use the orchestrator notes to choose the next smallest slice.
2. Assign backend, frontend, eval, or review work only when scopes are separable.
3. Record meaningful progress in `PROGRESS.md`.
4. Capture failed commands and debugging decisions in `LOGS.md`.
5. Keep final submission notes in `NOTES.md`.
